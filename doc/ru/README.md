## Адаптер automatic-pond-aeration для ioBroker

Этот адаптер **управляет системой аэрации пруда и контролирует её работу**. Воздушный
насос/компрессор подаёт воздух через клапаны (электромагнитные клапаны) к **до 8 точкам аэрации** в
пруду. Адаптер переключает эти клапаны по **расписанию**, по **циклическому кругу (round-robin)** или
по **групповой программе** и защищает насос с помощью **защитной блокировки**: пока насос работает,
всегда открыт хотя бы один клапан — иначе открывается **аварийный клапан** и (если насос доступен как
точка данных) насос отключается.

При желании он может контролировать **растворённый кислород**, **температуру воздуха и воды**, а также
**давление**, вычислять **астрономическое время** по вашей **геопозиции**, управлять оборудованием
**напрямую на ESP32** (без дополнительного экземпляра ioBroker) и приостанавливать выбранные точки
аэрации во время кормления, если установлен
[ioBroker.automatic-feeder](https://github.com/ssbingo/ioBroker.automatic-feeder).

> ⚠️ **Статус проекта: в разработке.** Модель конфигурации и полная модель точек данных готовы:
> адаптер проверяет вашу конфигурацию и соответствующим образом создаёт (и очищает) все свои объекты.
> Управляющая логика, аппаратные бэкенды и функции мониторинга добавляются шаг за шагом (по вехам).
> Для промышленного использования версия пока не предназначена.

## 1. Что делает адаптер

Аэрация пруда распределяет воздух от одного насоса на несколько распылителей/воздушных камней. Какие
точки получают воздух, решают **электромагнитные клапаны**. Этот адаптер решает, **когда** открывается
каждый клапан:

* **Расписание** — открывать точку/группу в настроенные временные окна по дням недели.
* **Циклический круг (round-robin)** — по очереди перебирать точки, каждая открыта в течение
  настраиваемого времени удержания.
* **Группы** — управлять несколькими точками совместно; групп **никогда не может быть больше, чем
  точек**.

Клапаны и насос можно приводить в действие либо через **существующие состояния ioBroker** (из любого
адаптера, предоставляющего переключатели), либо **напрямую на ESP32** с сопутствующей прошивкой.

## 2. Концепция безопасности

Воздушный компрессор **никогда не должен работать на полностью закрытые клапаны** (работа «в тупик»,
dead-heading) — это создаёт избыточное давление и может повредить насос. Поэтому:

* Пока насос работает, **всегда открыт хотя бы один клапан** (настраиваемый минимум).
* Если это не может быть гарантировано, **открывается аварийный клапан** и, если насосом можно
  управлять, **насос отключается**.
* Переключение клапанов работает по принципу **make-before-break** (следующий клапан открывается
  раньше, чем закрывается предыдущий), поэтому не бывает момента, когда все клапаны закрыты.

> 💡 **Рекомендация по подключению:** используйте **нормально открытый (NO)** аварийный клапан, чтобы
> он открывался при потере питания (отказоустойчивость). Когда оборудование работает на ESP32, та же
> блокировка выполняется и локально на устройстве, поэтому сбой сети или ioBroker не сможет повредить
> насос.

## 3. Требования

* Node.js ≥ 22
* js-controller ≥ 6.0.11, admin ≥ 7.6.20
* Один или несколько клапанов, доступных как состояния ioBroker, или ESP32 с сопутствующей прошивкой.

## 4. Установка

Установите адаптер из админки ioBroker (или, во время разработки, из репозитория GitHub) и создайте
экземпляр. Откройте настройки экземпляра, чтобы настроить его.

## 6. Объекты / точки данных

Адаптер создаёт свои точки данных из вашей конфигурации. Плейсхолдеры: `<n>` = индекс точки аэрации
(0–7), `<g>` = индекс группы. Объекты, помеченные **(w)**, — это команды, доступные для записи; все
остальные — доступные только для чтения значения состояния, обновляемые адаптером.

**Общее**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `info.connection` | boolean | `indicator.connected` | Адаптер работает / конфигурация корректна |
| `info.backend` | string | `text` | Активный аппаратный бэкенд (`iobroker` или `esp32`) |
| `info.activeMode` | string | `text` | Текущий режим работы |

**Управление (команды, доступные для записи)**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `control.enabled` | boolean (w) | `switch.enable` | Главное включение |
| `control.mode` | string (w) | `text` | Режим работы: `auto`, `manual` или `off` |
| `control.allOff` | boolean (w) | `button` | Закрыть все клапаны |
| `control.point.<n>.open` | boolean (w) | `switch` | Вручную открыть клапан точки `<n>` |
| `control.group.<g>.active` | boolean (w) | `switch` | Вручную активировать группу `<g>` |

**Точки аэрации** (по одному каналу на каждую настроенную точку, названному по имени точки)

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `aeration.point.<n>.valveState` | boolean | `indicator` | Клапан открыт |
| `aeration.point.<n>.active` | boolean | `indicator` | Точка сейчас аэрирует |
| `aeration.point.<n>.runtimeTodaySec` | number | `value` | Наработка за сегодня (секунды) |
| `aeration.point.<n>.runtimeTotalH` | number | `value` | Общая наработка (часы, для обслуживания) |
| `aeration.point.<n>.lastChange` | number | `value.time` | Отметка времени последнего изменения клапана |
| `aeration.point.<n>.error` | string | `text` | Последняя ошибка для этой точки |

**Группы**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `groups.<g>.members` | string | `json` | Индексы точек-участников |
| `groups.<g>.active` | boolean | `indicator` | Группа сейчас активна |

**Безопасность**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `safety.interlockActive` | boolean | `indicator.alarm` | Защитная блокировка сейчас активна |
| `safety.emergencyValve` | boolean | `indicator` | Аварийный клапан открыт |
| `safety.pumpRunning` | boolean | `indicator` | Насос работает |
| `safety.openValveCount` | number | `value` | Количество открытых клапанов |
| `safety.lastTripReason` | string | `text` | Причина последнего срабатывания блокировки |

**Датчики** (создаются только при включённом соответствующем мониторинге)

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `sensors.oxygen` | number | `value` | Растворённый кислород (mg/l) |
| `sensors.oxygenSaturation` | number | `value` | Насыщение кислородом (%) |
| `sensors.oxygenAlarm` | boolean | `indicator.alarm` | Кислород ниже нижнего порога |
| `sensors.airTemperature` | number | `value.temperature` | Температура воздуха (°C) |
| `sensors.waterTemperature` | number | `value.temperature` | Температура воды (°C) |
| `sensors.pressure` | number | `value.pressure` | Давление в системе (bar) |
| `sensors.pressureAlarm` | boolean | `indicator.alarm` | Давление вне диапазона |

**Астрономия и местоположение**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `astro.sunrise` / `astro.sunset` / `astro.solarNoon` | string | `text` | Время солнца для местоположения |
| `astro.isNight` | boolean | `indicator` | Сейчас ночь |
| `location.latitude` / `location.longitude` | number | `value.gps.*` | Определённые координаты |
| `location.resolvedAddress` | string | `text` | Определённый адрес |

**Связь с feeder** (создаётся только при включённой связи с feeder)

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `feeder.pauseActive` | boolean | `indicator` | Аэрация приостановлена на время кормления |
| `feeder.pauseUntil` | number | `value.time` | Пауза активна до |
| `feeder.lastFeedStart` | number | `value.time` | Последнее начало кормления |

**Статистика**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `statistics.compressorRuntimeTodayH` | number | `value` | Наработка компрессора за сегодня (часы) |
| `statistics.switchCyclesToday` | number | `value` | Циклов переключения клапанов за сегодня |

Когда точка, группа или датчик удаляются из конфигурации, их объекты очищаются автоматически.
