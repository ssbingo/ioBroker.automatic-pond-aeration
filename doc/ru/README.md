## Адаптер automatic-pond-aeration для ioBroker

Этот адаптер **управляет системой аэрации пруда и контролирует её работу**. Воздушный
насос/компрессор подаёт воздух через клапаны (электромагнитные клапаны) к **до 8 точкам аэрации** в
пруду. Адаптер переключает эти клапаны по **расписанию**, по **циклическому кругу (round-robin)** или
по **групповой программе** и защищает насос с помощью **защитной блокировки**: пока насос работает,
всегда открыт хотя бы один клапан — иначе открывается **аварийный клапан** и (если насос доступен как
точка данных) насос отключается.

При желании он может контролировать **растворённый кислород**, **температуру воздуха и воды**, а также
**давление**, вычислять **астрономическое время** по вашей **геопозиции**, управлять оборудованием
**напрямую на ESP32** (без дополнительного экземпляра ioBroker) и приостанавливать выбранные точки
аэрации во время кормления, если установлен
[ioBroker.automatic-feeder](https://github.com/ssbingo/ioBroker.automatic-feeder).

## 1. Что делает адаптер

Аэрация пруда распределяет воздух от одного насоса на несколько распылителей/воздушных камней. Какие
точки получают воздух, решают **электромагнитные клапаны**. Этот адаптер решает, **когда** открывается
каждый клапан:

* **Расписание** — открывать точку/группу в настроенные временные окна по дням недели.
* **Циклический круг (round-robin)** — по очереди перебирать точки, каждая открыта в течение
  настраиваемого времени удержания.
* **Группы** — управлять несколькими точками совместно; групп **никогда не может быть больше, чем
  точек**.

Клапаны и насос приводятся в действие **либо** через **существующие состояния ioBroker** (из любого
адаптера, предоставляющего переключатели), **либо напрямую на выделенном контроллере ESP32**, на
котором работает эталонная прошивка, — без отдельного экземпляра ioBroker. Вы выбираете это в разделе
**Аппаратный бэкенд** (вкладка «Общее»); см. [Конфигурация → Общее](#общее).

## 2. Концепция безопасности

Воздушный компрессор **никогда не должен работать на полностью закрытые клапаны** (работа «в тупик»,
dead-heading) — это создаёт избыточное давление и может повредить насос. Поэтому:

* Пока насос работает, **всегда открыт хотя бы один клапан** (настраиваемый минимум).
* Если это не может быть гарантировано, **открывается аварийный клапан** и, если насосом можно
  управлять, **насос отключается**.
* Переключение клапанов работает по принципу **make-before-break** (следующий клапан открывается
  раньше, чем закрывается предыдущий), поэтому не бывает момента, когда все клапаны закрыты.

> 💡 **Рекомендация по подключению:** используйте **нормально открытый (NO)** аварийный клапан, чтобы
> он открывался при потере питания (отказоустойчивость). Когда оборудование работает на ESP32, та же
> блокировка выполняется и локально на устройстве, поэтому сбой сети или ioBroker не сможет повредить
> насос.

## 3. Требования

* Node.js ≥ 22
* js-controller ≥ 6.0.11, admin ≥ 7.6.20
* Один или несколько клапанов, доступных как состояния ioBroker (например, адаптер реле/умной розетки).

## 4. Установка

Установите адаптер из админки ioBroker (или, во время разработки, из репозитория GitHub) и создайте
экземпляр. Откройте настройки экземпляра, чтобы настроить его.

## 5. Конфигурация

Страница настроек разбита на вкладки. Заполнять всё необязательно — только те части, которые вы
используете.

### Общее
- **Главное включение** — переключатель вкл/выкл для всего адаптера. Когда выключено, ничего не
  управляется.
- **Холостой прогон (dry-run, только журнал, без переключения оборудования)** — вся управляющая
  логика работает и точки данных обновляются, но команды на клапаны/насос только записываются в
  журнал (`[DRY-RUN] would …`) вместо реальных состояний. Идеально для пусконаладки и проверки
  конфигурации перед подключением оборудования.
- **Аппаратный бэкенд** — `Существующие состояния ioBroker` (по умолчанию) управляет вашими
  клапанами/насосом через состояния других адаптеров. `ESP32 (напрямую)` общается по HTTP с эталонной
  прошивкой на Waveshare ESP32-S3-POE-ETH-8DI-8RO. Прошейте прошивку прямо в браузере со
  [страницы прошивки](https://ssbingo.github.io/pond-aeration-flash/) (Chrome/Edge, без
  дополнительного ПО), затем задайте **host/IP** и сопоставьте **реле аварийного клапана** и **реле
  насоса** (0–7); точки аэрации используют канал реле, заданный для каждой точки. Адаптер отправляет
  конфигурацию безопасности и сигнал heartbeat, чтобы встроенная в устройство защита прошивки берегла
  пруд, даже если ioBroker недоступен.
  - **Автономное расписание (работает без ioBroker)** *(только ESP32, необязательно)* – если
    включено, адаптер также отправляет ваши расписания на устройство; если соединение прерывается,
    ESP32 продолжает выполнять их самостоятельно по своим часам NTP (защитная блокировка dead-head
    по-прежнему действует). Циклическая последовательность остаётся на стороне адаптера.
  - **Совместимость прошивки** – адаптер и прошивка сопоставляются по **версии протокола** (жёсткий
    контракт), а не по точным номерам релизов. Эта версия адаптера говорит на **протоколе 1** и
    **рекомендует прошивку v1.4.0** (минимум v1.0.0); админка показывает это и ссылается на релизы.
    При подключении версия устройства и флаг совместимости публикуются как `info.deviceFirmware` и
    `info.firmwareCompatible`, а любое несоответствие протокола записывается в журнал. См. таблицу
    совместимости в [руководстве](../../docs/manual/pond-aeration-manual.en.pdf) / репозитории
    прошивки.
  - **Лицензирование** *(только если ваша прошивка включает необязательный слой лицензирования)* –
    устройство работает на одном из уровней: **free** (только мониторинг), **community** (управление
    реле) или **pro** (+ автономное standalone-расписание); безопасность (отказоустойчивость,
    аварийный клапан, защитная блокировка от работы «в тупик», ручные кнопки) всегда активна
    независимо от уровня. Новое устройство работает в полном объёме (**pro**) в течение пробного
    периода, затем возвращается к free, пока на странице устройства `/license` не будет введён ключ
    активации. Адаптер показывает статус в `info.licenseTier` / `info.licenseTrialDaysLeft` /
    `info.deviceCode`; если у устройства **нет лицензии на управление**, мониторинг продолжает
    работать, а управление пропускается (см. `info.licenseControlBlocked`). Публичная прошивка без
    этого слоя не затрагивается.
    *Примечание о повторной прошивке:* ключ активации хранится на ESP и **стирается при повторной
    прошивке через браузерный установщик** (начинается новый пробный период). **Код устройства
    выводится из аппаратуры и никогда не меняется**, поэтому **тот же ключ активации можно просто
    ввести повторно** — новый ключ не нужен. **Обновление прошивки через страницу Update устройства**
    (обновление онлайн одним кликом или загрузка файла) сохраняет активацию и все настройки; сбрасывает
    её только установщик.
  - **Зеркалирование датчиков** – при каждом опросе адаптер также отправляет ваши настроенные точки
    данных датчиков (кислород, температура воды/воздуха, давление) на устройство, так что они
    появляются в **собственном веб-интерфейсе ESP** (с пометкой *(ioBroker)*) — даже для датчиков,
    которые являются только состояниями ioBroker и не подключены к ESP. Физически подключённый к ESP
    датчик сохраняет приоритет; отправленные значения исчезают через несколько минут. Требуется
    прошивка ≥ 1.1.7.
- **Интервал опроса (s)** — как часто опрашивается статус бэкенда (например, `30`).

### Точки аэрации
Ядро конфигурации. Добавьте **до 8** точек; каждая точка — это один клапан. Для каждой точки:
- **Имя** — например, `Pier`, `Deep zone`. При бэкенде **ESP32** это имя также **отображается в
  собственном веб-интерфейсе устройства** (на релейном канале точки) — это **лицензируемая функция**
  (начиная с уровня **community**). `Ch 7 = Notventil` (аварийный клапан) и `Ch 8 = Pumpe` (насос) —
  это фиксированные подписи. См. [Имена в веб-интерфейсе ESP32](#имена-в-веб-интерфейсе-esp32).
- **Включено** — включить эту точку в управление.
- **Бэкенд** — `ioBroker` (внешнее состояние) или `ESP32` (канал реле на устройстве). Опция `ESP32`
  появляется только тогда, когда **Аппаратный бэкенд** (вкладка «Общее») установлен в `ESP32
  (напрямую)`.
- **Состояние клапана / канал** — для бэкенда ioBroker выберите состояние-переключатель, которое
  открывает клапан (через браузер объектов). Для бэкенда ESP32 выберите **канал реле** из
  выпадающего списка: каналы, управляющие **насосом** и **аварийным клапаном**, показаны как
  *зарезервированные*, а каналы, уже занятые другой точкой, — как *используемые*, так что вы можете
  выбрать только свободный. Когда свободных каналов не остаётся, добавляйте дополнительные точки как
  **состояния ioBroker** через столбец «Бэкенд».
- **Кнопка ручного переопределения** *(опционально)* — физическая кнопка для каждой точки (например,
  цифровой вход ESP32 или любое булево состояние). Работает как **переключатель (toggle)**: одно
  нажатие принудительно **включает точку с приоритетом над автоматическим управлением**
  (расписание/последовательность/зима/кислород) и даже над паузой feeder — *отменить его могут только
  главный выключатель или срабатывание защиты*. Нажмите ещё раз, чтобы отпустить. (Планируются другие
  режимы кнопки; поле для них подготовлено.) Кнопка доступна только для **клапана аэрации** — точка,
  которая находится на релейном канале ESP32 для **насоса** или **аварийного клапана**, не может её
  иметь (опция недоступна). При бэкенде ESP32 кнопка, нажатая **на устройстве**, отражается обратно в
  ioBroker (`aeration.point.<n>.buttonOn`) и получает тот же приоритет.
- **Имя кнопки** *(бэкенд ESP32, необязательно)* — понятное имя для кнопки переопределения этой точки,
  отображаемое в веб-интерфейсе устройства (см. ниже). Пусто → кнопка показывает имя точки.

#### Имена в веб-интерфейсе ESP32

*(Бэкенд ESP32, **лицензируемая функция** — доступна начиная с уровня **community**.)* Задайте своим
каналам и кнопкам понятные имена, которые отображаются на собственных веб-страницах устройства вместо
`Ch 1…8` / `DI 1…8`:

- Адаптер **отправляет имя каждой точки аэрации** на её релейный канал (Ch 1–6), а необязательное
  **имя кнопки** каждой точки — на соответствующий цифровой вход (DI 1–8).
- **Ch 7 = Notventil** (аварийный клапан) и **Ch 8 = Pumpe** (насос) **фиксированы** и не могут быть
  переименованы.
- **Автономно (без адаптера):** те же имена можно редактировать **на устройстве** в разделе
  *Settings → Namen (Kanäle & Taster)*; они хранятся на ESP (NVS); когда адаптер подключён, он
  перезаписывает их именами, настроенными здесь.
- На **free** (нелицензированной) прошивке имена игнорируются, и на страницах отображаются подписи по
  умолчанию `Ch`/`DI`.

### Группы
Объедините точки в группы, чтобы переключать их вместе (например, одна кнопка открывает несколько
распылителей). Задайте группе имя и отметьте её точки-участники. **Групп никогда не может быть больше,
чем точек.**

### Управление
- **Циклический round-robin** — по очереди перебирать точки, каждая открыта в течение **времени
  удержания** (секунды).
  - **Последовательность (точки и группы)** — при желании задайте **упорядоченный цикл шагов**, где
    каждый шаг нацелен на одну **точку или целую группу** и может иметь собственное время удержания.
    Это позволяет выполнять, например, *группа 1 → группа 3 → точка 1 → …* и свободно **смешивать**
    точки и группы. Меняйте порядок шагов стрелками вверх/вниз в админке. Оставьте последовательность
    пустой, чтобы вернуться к простому round-robin по всем точкам.
- **Расписания** — открывать выбранные точки/группы в течение временного окна по дням недели.
  **С**/**По** выбираются с помощью **выбора времени на часах** (час/минута, 24 ч; окна через ночь,
  такие как `22:00`–`06:00`, поддерживаются). Активное расписание имеет **приоритет над round-robin /
  последовательностью**.
- **Зимний / незамерзающий режим** — в течение настроенного сезона (**С**/**По** выбираются из
  **календаря** — учитываются только **день и месяц**, повторяется каждый год, например 1 нояб. –
  15 мар., с переходом через новый год) выбранные точки принудительно
  включаются, чтобы поддерживать открытую незамерзающую лунку. При желании отметьте **Только когда
  холодно (защита от мороза)** и задайте **порог температуры воздуха**, чтобы пруд аэрировался только
  при фактическом заморозке (для этого нужен мониторинг температуры воздуха). Оставьте
  **Поддерживаемые открытыми точки** пустыми, чтобы аэрировать весь пруд. Зимний режим работает в
  режиме `auto` и, как и любая программа, уступает защитной блокировке и паузе feeder.

### Датчики
Опциональный мониторинг. Для каждого датчика отметьте **Включено** и выберите **исходное состояние**:
- **Растворённый кислород** — с нижним порогом (поднимает `sensors.oxygenAlarm`), целевым значением и
  гистерезисом; **насыщение кислородом %** вычисляется по температуре воды.
  - **Замкнутый контур по кислороду** — при включении адаптер **принудительно включает аэрацию**, пока
    кислород ниже нижнего порога, и удерживает её, пока значение не восстановится до целевого (или до
    `low + hysteresis`, если целевое не задано). Оставьте **Усиливаемые точки** пустыми, чтобы усилить
    аэрацию всего пруда. Как и зимний режим, контур работает в режиме `auto` и уступает защитной
    блокировке и паузам feeder.
- **Температура воздуха/воды**.
- **Давление** — с мин/макс (выход за пределы поднимает `sensors.pressureAlarm`).

### Местоположение
Нужно для астрономического времени (восход/закат/ночь).
- **Источник местоположения** — `Системное местоположение ioBroker` (использует ваши системные
  координаты) или `Собственное местоположение`. Для собственного местоположения введите адрес и
  нажмите **Поиск** (по запросу геокодируется через OpenStreetMap/Nominatim) либо кликните/перетащите
  маркер на карте.

### Feeder
Приостанавливать выбранные точки, пока
[ioBroker.automatic-feeder](https://github.com/ssbingo/ioBroker.automatic-feeder) кормит, чтобы корм
не разносило.
- Выберите **экземпляр feeder** (обнаруживается автоматически) и отметьте отслеживаемые
  **переключатели feeder**.
- **Режим длительности** — `Измерять` отслеживает переключатель (пауза = кормление + смещение, без
  предварительного знания длительности кормления); `Импульс` использует фиксированную длительность
  кормления.
- **Смещение (s)** — дополнительная пауза после окончания кормления. **Оно должно быть не меньше
  среднего времени, которое нужно животным, чтобы поесть** (пример: 15 s кормления + 60 s смещения ⇒
  75 s приостановленной аэрации).
- **Затрагиваемые точки** — какие точки приостанавливаются во время кормления.

### Безопасность
Каждое поле на этой вкладке снабжено **пояснением в админке** о том, что оно делает и каков его
эффект, — прочитайте их, ведь это вкладка, где неверное значение важнее всего.
- **Мин. число открытых клапанов при работающем насосе** — защита от работы «в тупик» (по умолчанию
  `1`).
- **Интервал watchdog (s)** и **перекрытие make-before-break (s)**.
- **Насос** — управляем ли он, **сигнал** насоса, а также минимальные времена вкл/выкл против частого
  тактования. Когда насосом можно управлять, адаптер **управляет насосом так, чтобы он следовал за
  потребностью в аэрации** — он работает, пока открыто хотя бы минимальное число клапанов, и
  отключается, когда пруд простаивает или при срабатывании защиты от работы «в тупик» (с соблюдением
  минимальных времён вкл/выкл); когда управлять им *нельзя*, насос только наблюдается, и его защищает
  один лишь аварийный клапан. *При бэкенде **ESP32** сигнал насоса — это
  **канал реле ESP32**, тот самый, что задан в «Общее → Аппаратный бэкенд», показанный здесь, чтобы
  две вкладки никогда не могли противоречить друг другу; при бэкенде **ioBroker** это состояние
  ioBroker.*
- **Аварийный клапан** — его **сигнал**, является ли он **нормально открытым** (fail-safe), **тип**
  клапана (электромагнитный или моторизованный шаровой кран) и, для моторного клапана, его **время
  хода**. *При бэкенде ESP32 сигнал — это точно так же канал реле аварийного клапана ESP32 (тот же,
  что в «Общее»).*

### Уведомления
Включите уведомления и выберите **экземпляр messaging** (любой адаптер типа `messaging`, например
Telegram или Pushover), затем **отметьте, какие события** должны отправлять сообщение:
- **Защитная блокировка** — когда блокировка от работы «в тупик» срабатывает или снимается;
- **Кислородная тревога** — когда растворённый кислород падает слишком низко или восстанавливается;
- **Тревога по давлению** — когда давление выходит за пределы диапазона или возвращается в него.

На каждом фронте (срабатывание и снятие) отправляется короткий локализованный текст. Если не отмечено
ни одно событие, ничего не отправляется.

## 6. Объекты / точки данных

Адаптер создаёт свои точки данных из вашей конфигурации. Плейсхолдеры: `<n>` = индекс точки аэрации
(0–7), `<g>` = индекс группы. Объекты, помеченные **(w)**, — это команды, доступные для записи; все
остальные — доступные только для чтения значения состояния, обновляемые адаптером.

**Общее**

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `info.connection` | boolean | `indicator.connected` | Адаптер работает / конфигурация корректна |
| `info.backend` | string | `text` | Активный аппаратный бэкенд (`iobroker` или `esp32`) |
| `info.activeMode` | string | `text` | Текущий режим работы |
| `info.dryRun` | boolean | `indicator` | Активен холостой прогон (оборудование не переключается) |

**Бэкенд ESP32 (info)** (только при аппаратном бэкенде ESP32)

| Объект | Тип | Роль | Описание |
|--------|-----|------|----------|
| `info.deviceFirmware` | string | `text` | Версия прошивки, сообщённая ESP32 |
| `info.firmwareCompatible` | boolean | `indicator` | Протокол прошивки совместим с этим адаптером |
| `info.licenseTier` | string | `text` | Активный уровень лицензии: `free` (мониторинг), `community` (управление реле) или `pro` (+ автономное расписание); пусто, если прошивка без лицензирования |
| `info.licenseTrialDaysLeft` | number | `value` | Осталось дней пробного периода лицензии (0 = пробный период не активен) |
| `info.deviceCode` | string | `text` | Код устройства — сообщите его при разблокировке, чтобы получить ключ активации |
| `info.licenseControlBlocked` | boolean | `indicator` | Устройство отклонило команду управления (нет лицензии на управление) |

**Управление (команды, доступные для записи)**
