## Адаптер automatic-pond-aeration для ioBroker

Цей адаптер **керує системою аерації ставка та контролює її роботу**. Повітряний насос/компресор подає
повітря через клапани (електромагнітні клапани) до **до 8 точок аерації** у ставку. Адаптер перемикає
ці клапани за **розкладом**, за **циклічним колом (round-robin)** або за **груповою програмою** та
захищає насос за допомогою **захисного блокування**: доки насос працює, завжди залишається відкритим
щонайменше один клапан – інакше відкривається **аварійний клапан** і (якщо насос доступний як точка
даних) насос вимикається.

За бажанням він може контролювати **розчинений кисень**, **температуру повітря та води** й **тиск**,
обчислювати **астрономічний час** за вашою **геопозицією**, керувати обладнанням **безпосередньо на
ESP32** (без додаткового екземпляра ioBroker) та призупиняти вибрані точки аерації під час годування,
якщо встановлено [ioBroker.automatic-feeder](https://github.com/ssbingo/ioBroker.automatic-feeder).

## 1. Що робить адаптер

Аерація ставка розподіляє повітря від одного насоса на кілька розпилювачів/повітряних каменів. Які
точки отримують повітря, вирішують **електромагнітні клапани**. Цей адаптер вирішує, **коли**
відкривається кожен клапан:

* **Розклад** – відкривати точку/групу протягом налаштованих часових вікон за днями тижня.
* **Циклічне коло (round-robin)** – по черзі перемикати точки, кожна відкрита протягом
  налаштовуваного часу утримання.
* **Групи** – керувати кількома точками разом; груп **ніколи не може бути більше, ніж точок**.

Клапани й насос приводяться в дію **або** через **наявні стани ioBroker** (з будь-якого адаптера, що
надає перемикачі), **або безпосередньо на виділеному контролері ESP32**, на якому працює еталонна
прошивка — без додаткового екземпляра ioBroker. Ви обираєте це в розділі **Апаратний бекенд** (вкладка
«Загальне»); див. [Конфігурація → Загальне](#загальне).

## 2. Концепція безпеки

Повітряний компресор **ніколи не повинен працювати на повністю закриті клапани** (робота «в глухий
кут», dead-heading) – це створює надлишковий тиск і може пошкодити насос. Тому:

* Доки насос працює, **завжди відкритий щонайменше один клапан** (налаштовуваний мінімум).
* Якщо цього не можна гарантувати, **відкривається аварійний клапан** і, якщо насосом можна керувати,
  **насос вимикається**.
* Перемикання клапанів працює за принципом **make-before-break** (наступний клапан відкривається
  раніше, ніж закривається попередній), тож не буває моменту, коли всі клапани закриті.

> 💡 **Рекомендація щодо підключення:** використовуйте **нормально відкритий (NO)** аварійний клапан,
> щоб він відкривався при зникненні живлення (відмовостійкість). Коли обладнання працює на ESP32, те
> саме блокування виконується й локально на пристрої, тож збій мережі або ioBroker не зможе пошкодити
> насос.

## 3. Передумови

* Node.js ≥ 22
* js-controller ≥ 6.0.11, admin ≥ 7.6.20
* Один або кілька клапанів, доступних як стани ioBroker (наприклад, адаптер реле/розумної розетки).

## 4. Встановлення

Встановіть адаптер з адмінки ioBroker (або, під час розробки, з репозиторію GitHub) і створіть
екземпляр. Відкрийте налаштування екземпляра, щоб налаштувати його.

## 5. Конфігурація

Сторінка налаштувань поділена на вкладки. Заповнювати все необов'язково — лише ті частини, які ви
використовуєте.

### Загальне
- **Головне ввімкнення** — перемикач увімк./вимк. усього адаптера. Коли вимкнено, нічим не керується.
- **Сухий прогін (лише журнал, без перемикання обладнання)** — уся керуюча логіка працює й точки
  даних оновлюються, але команди клапанів/насоса лише записуються в журнал (`[DRY-RUN] would …`)
  замість реальних станів. Ідеально для введення в експлуатацію та перевірки конфігурації перед
  підключенням обладнання.
- **Апаратний бекенд** — `Наявні стани ioBroker` (за замовчуванням) керує вашими клапанами/насосом
  через стани інших адаптерів. `ESP32 (напряму)` спілкується через HTTP з еталонною прошивкою на
  Waveshare ESP32-S3-POE-ETH-8DI-8RO. Прошийте прошивку прямо в браузері зі [сторінки
  прошивки](https://ssbingo.github.io/pond-aeration-flash/) (Chrome/Edge, без додаткового ПЗ), потім
  задайте **host/IP** і зіставте **реле аварійного клапана** та **реле насоса** (0–7); точки аерації
  використовують канал реле, заданий для кожної точки. Адаптер надсилає конфігурацію безпеки та сигнал
  heartbeat, щоб вбудований у пристрій захист прошивки оберігав ставок, навіть якщо ioBroker
  недоступний.
  - **Автономний розклад (працює без ioBroker)** *(лише ESP32, необов'язково)* – коли ввімкнено,
    адаптер також надсилає ваші розклади на пристрій; якщо з'єднання переривається, ESP32 самостійно
    продовжує їх виконувати за власним годинником NTP (захисне блокування dead-head продовжує
    діяти). Циклічна послідовність залишається на боці адаптера.
  - **Сумісність прошивки** – адаптер і прошивка зіставляються за **версією протоколу** (жорсткий
    контракт), а не за точними номерами релізів. Ця версія адаптера говорить **протоколом 1** і
    **рекомендує прошивку v1.6.0** (мінімум v1.0.0); адмінка показує це та посилається на релізи. Під
    час підключення версія пристрою та прапорець сумісності публікуються як `info.deviceFirmware` та
    `info.firmwareCompatible`, а будь-яка невідповідність протоколу записується в журнал. Див. таблицю
    сумісності в [посібнику](../../docs/manual/pond-aeration-manual.en.pdf) / репозиторії прошивки.
  - **Ліцензування** *(лише якщо ваша прошивка постачається з опційним оверлеєм ліцензування)* – пристрій
    працює на певному рівні: **free** (лише моніторинг), **community** (керування реле) або **pro**
    (+ автономний розклад); безпека (відмовостійкість, аварійний клапан, блокування від роботи
    «в глухий кут», ручні кнопки) завжди активна незалежно від цього. Новий пристрій працює на повну
    (**pro**) протягом пробного періоду, а потім повертається до free, доки на сторінці пристрою
    `/license` не буде введено ключ активації. Адаптер показує статус у `info.licenseTier` /
    `info.licenseTrialDaysLeft` / `info.deviceCode`; якщо пристрій **не має ліцензії на керування**,
    моніторинг продовжує працювати, а керування пропускається (див. `info.licenseControlBlocked`).
    Публічна прошивка без оверлея це не зачіпає.
    *Примітка щодо повторного прошивання:* ключ активації зберігається на ESP і **стирається під час
    повторного прошивання через браузерний інсталятор** (розпочинається новий пробний період). **Код
    пристрою є апаратно похідним і ніколи не змінюється**, тож **можна просто повторно ввести той
    самий ключ активації** — новий ключ не потрібен. Оновлення прошивки **через сторінку Update
    пристрою** (оновлення онлайн одним кліком або завантаження файлу) зберігає активацію та всі
    налаштування; лише інсталятор скидає її.
  - **Дзеркалення датчиків** – при кожному опитуванні адаптер додатково надсилає ваші налаштовані
    точки даних датчиків (кисень, температура води/повітря, тиск) на пристрій, тож вони з'являються на
    **власному вебінтерфейсі ESP** (позначені *(ioBroker)*) — навіть для датчиків, які є лише станами
    ioBroker і не підключені до ESP. Фізично підключений до ESP датчик зберігає пріоритет; надіслані
    значення зникають за кілька хвилин. Потрібна прошивка ≥ 1.1.7.
- **Інтервал опитування (s)** — як часто опитується статус бекенда (наприклад, `30`).

### Точки аерації
Серце конфігурації. Додайте **до 8** точок; кожна точка — це один клапан. Для кожної точки:
- **Ім'я** — наприклад, `Pier`, `Deep zone`. За бекенду **ESP32** це ім'я також **відображається у
  власному вебінтерфейсі пристрою** (на релейному каналі точки) — це **ліцензована функція** (починаючи
  з рівня **community**). `Ch 7 = Notventil` (аварійний клапан) і `Ch 8 = Pumpe` (насос) — це фіксовані
  підписи. Див. [Імена у вебінтерфейсі ESP32](#імена-у-вебінтерфейсі-esp32).
- **Увімкнено** — включити цю точку до керування.
- **Бекенд** — `ioBroker` (зовнішній стан) або `ESP32` (канал реле на пристрої). Опція `ESP32`
  з'являється лише тоді, коли **Апаратний бекенд** (вкладка «Загальне») має значення `ESP32 (напряму)`.
- **Стан клапана / канал** — для бекенда ioBroker виберіть стан-перемикач, який відкриває клапан
  (через браузер об'єктів). Для бекенда ESP32 виберіть **канал реле** з випадного списку: канали, що
  керують **насосом** і **аварійним клапаном**, показані як *зарезервовані*, а канали, вже зайняті
  іншою точкою, — як *використовуються*, тож вибрати можна лише вільний. Коли не залишилося вільного
  каналу, додайте подальші точки як **стани ioBroker** через стовпець «Бекенд».
- **Кнопка ручного перевизначення** *(необов'язково)* — фізична кнопка для кожної точки (наприклад,
  цифровий вхід ESP32 або будь-який булевий стан). Працює як **перемикач (toggle)**: одне натискання
  примусово **вмикає точку з пріоритетом над автоматичним керуванням**
  (розклад/послідовність/зима/кисень) і навіть над паузою feeder — *скасувати його можуть лише
  головний вимикач або спрацювання захисту*. Натисніть ще раз, щоб відпустити. (Плануються інші
  режими кнопки; поле для них підготовлено.) Кнопка доступна лише для **клапана аерації** — точка, що
  розташована на релейному каналі ESP32 для **насоса** або **аварійного клапана**, не може її мати
  (опція неактивна). За бекенду ESP32 кнопка, натиснута **на пристрої**, відображається назад в
  ioBroker (`aeration.point.<n>.buttonOn`) і отримує той самий пріоритет.
- **Ім'я кнопки** *(бекенд ESP32, необов'язково)* — зрозуміле ім'я для кнопки перевизначення цієї
  точки, що відображається у вебінтерфейсі пристрою (див. нижче). Порожньо → кнопка показує ім'я точки.

#### Імена у вебінтерфейсі ESP32

*(Бекенд ESP32, **ліцензована функція** — доступна починаючи з рівня **community**.)* Дайте своїм
каналам і кнопкам зрозумілі імена, які відображаються на власних вебсторінках пристрою замість
`Ch 1…8` / `DI 1…8`:

- Адаптер **надсилає ім'я кожної точки аерації** на її релейний канал (Ch 1–6), а необов'язкове **ім'я
  кнопки** кожної точки — на відповідний цифровий вхід (DI 1–8).
- **Ch 7 = Notventil** (аварійний клапан) і **Ch 8 = Pumpe** (насос) **фіксовані** й не можуть бути
  перейменовані.
- **Автономно (без адаптера):** ті самі імена можна редагувати **на пристрої** в розділі *Settings →
  Namen (Kanäle & Taster)*; вони зберігаються на ESP (NVS); коли адаптер підключений, він перезаписує
  їх іменами, налаштованими тут.
- На **free** (неліцензованій) прошивці імена ігноруються, і на сторінках відображаються типові
  підписи `Ch`/`DI`.

### Групи
Об'єднуйте точки в групи, щоб перемикати їх разом (наприклад, одна кнопка відкриває кілька
розпилювачів). Дайте групі ім'я та позначте її точки-учасники. **Груп ніколи не може бути більше, ніж
точок.**

### Керування
- **Циклічний round-robin** — по черзі перемикати точки, кожна відкрита протягом **часу утримання**
  (секунди).
  - **Послідовність (точки та групи)** — за бажанням задайте **упорядкований цикл кроків**, де кожен
    крок націлений на одну **точку або цілу групу** і може мати власний час утримання. Це дозволяє
    виконувати, наприклад, *група 1 → група 3 → точка 1 → …* і вільно **змішувати** точки та групи.
    Змінюйте порядок кроків стрілками вгору/вниз в адмінці. Залиште послідовність порожньою, щоб
    повернутися до простого round-robin по всіх точках.
- **Розклади** — відкривати вибрані точки/групи протягом часового вікна за днями тижня. **Від**/**До**
  вибираються за допомогою **вибору часу на годиннику** (година/хвилина, 24 год; вікна через ніч,
  як-от `22:00`–`06:00`, підтримуються). Активний розклад має **пріоритет над round-robin /
  послідовністю**.
- **Зимовий / незамерзаючий режим** — протягом налаштованого сезону (**Початок**/**Кінець** вибираються
  з **календаря** — враховуються лише **день і місяць**, повторюється щороку, наприклад 1 лист. –
  15 бер., з переходом через новий рік) вибрані точки
  примусово вмикаються, щоб тримати ополонку відкритою. За бажанням позначте **Лише коли холодно
  (захист від морозу)** і задайте **поріг температури повітря**, щоб ставок аерувався лише за
  фактичного замерзання (для цього потрібен моніторинг температури повітря). Залиште **Точки, що
  тримаються відкритими** порожнім, щоб аерувати весь ставок. Зимовий режим працює в режимі роботи
  `auto` і, як і кожна програма, все одно поступається захисному блокуванню та паузі feeder.

### Датчики
Опційний моніторинг. Для кожного датчика позначте **Увімкнено** та виберіть **джерельний стан**:
- **Розчинений кисень** — з нижнім порогом (піднімає `sensors.oxygenAlarm`), цільовим значенням і
  гістерезисом; **насичення киснем %** обчислюється з температури води.
  - **Кисневий замкнений контур** — коли ввімкнено, адаптер **примусово вмикає аерацію**, доки
    кисень нижче нижнього порога, і тримає її ввімкненою, доки він не відновиться до цільового
    значення (або `low + hysteresis`, якщо ціль не задано). Залиште **Форсовані точки** порожнім,
    щоб форсувати весь ставок. Як і зимовий режим, контур працює в режимі `auto` і поступається
    захисту та паузам feeder.
- **Температура повітря/води**.
- **Тиск** — з мін/макс (вихід за межі піднімає `sensors.pressureAlarm`).

### Розташування
Потрібне для астрономічного часу (схід/захід сонця/ніч).
- **Джерело розташування** — `Системне розташування ioBroker` (використовує координати вашої системи)
  або `Власне розташування`. Для власного розташування введіть адресу й натисніть **Пошук** (за
  запитом геокодується через OpenStreetMap/Nominatim) або клацніть/перетягніть маркер на мапі.

### Feeder
Призупиняти вибрані точки, поки
[ioBroker.automatic-feeder](https://github.com/ssbingo/ioBroker.automatic-feeder) годує, щоб корм не
розносило.
- Виберіть **екземпляр feeder** (виявляється автоматично) і позначте відстежувані **перемикачі
  feeder**.
- **Режим тривалості** — `Вимірювати` стежить за перемикачем (пауза = годування + зсув, без
  попереднього знання тривалості годування); `Імпульс` використовує фіксовану тривалість годування.
- **Зсув (s)** — додаткова пауза після завершення годування. **Він має бути щонайменше рівним
  середньому часу, потрібному тваринам, щоб поїсти** (приклад: 15 s годування + 60 s зсуву ⇒ 75 s
  призупиненої аерації).
- **Задіяні точки** — які точки призупиняються під час годування.

### Безпека
Кожне поле на цій вкладці має **пояснення в адмінці** про те, що воно робить і який його ефект, —
прочитайте їх, адже це вкладка, де неправильне значення важить найбільше.
- **Мін. відкритих клапанів при працюючому насосі** — захист від роботи «в глухий кут» (за
  замовчуванням `1`).
- **Інтервал watchdog (s)** та **перекриття make-before-break (s)**.
- **Насос** — чи він керований, **сигнал** насоса, а також мінімальні часи ввімк./вимк. проти надто
  частого тактування. Коли насосом можна керувати, адаптер **керує насосом так, щоб він слідував за
  потребою в аерації** — він працює, доки відкрито щонайменше мінімальну кількість клапанів, і
  вимикається, коли ставок простоює або при спрацюванні захисту від роботи «в глухий кут» (з
  дотриманням мінімальних часів ввімк./вимк.); коли керувати ним *не можна*, насос лише спостерігається,
  і його захищає сам лише аварійний клапан. *За бекенду **ESP32** сигнал насоса —
  це **канал реле ESP32** — той самий, що заданий у розділі «Загальне → Апаратний бекенд», показаний
  тут, щоб дві вкладки ніколи не могли суперечити одна одній; за бекенду **ioBroker** це стан
  ioBroker.*
- **Аварійний клапан** — його **сигнал**, чи він **нормально відкритий** (fail-safe), **тип** клапана
  (електромагнітний або моторизований кульовий кран) і, для моторного клапана, його **час ходу**. *За
  бекенду ESP32 сигнал так само є каналом реле аварійного клапана ESP32 (як у розділі «Загальне»).*

### Сповіщення
Увімкніть сповіщення та виберіть **екземпляр messaging** (будь-який адаптер типу `messaging`,
наприклад Telegram або Pushover), потім **позначте, які події** мають надсилати повідомлення:
- **Захисне блокування** — коли блокування від роботи «в глухий кут» спрацьовує або знімається;
- **Кисневий аварійний сигнал** — коли розчинений кисень падає надто низько або відновлюється;
- **Аварійний сигнал тиску** — коли тиск виходить за межі діапазону або повертається в нього.

На кожному фронті (спрацювання та зняття) надсилається короткий локалізований текст. Якщо не позначено
жодної події, нічого не надсилається.

## 6. Об'єкти / точки даних

Адаптер створює свої точки даних із вашої конфігурації. Заповнювачі: `<n>` = індекс точки аерації
(0–7), `<g>` = індекс групи. Об'єкти, позначені **(w)**, — це команди, доступні для запису; усі інші
— доступні лише для читання значення стану, які оновлює адаптер.

**Загальне**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `info.connection` | boolean | `indicator.connected` | Адаптер працює / конфігурація дійсна |
| `info.backend` | string | `text` | Активний апаратний бекенд (`iobroker` або `esp32`) |
| `info.activeMode` | string | `text` | Поточний режим роботи |
| `info.dryRun` | boolean | `indicator` | Сухий прогін активний (обладнання не перемикається) |

**Бекенд ESP32 (info)** (лише з апаратним бекендом ESP32)

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `info.deviceFirmware` | string | `text` | Версія прошивки, повідомлена ESP32 |
| `info.firmwareCompatible` | boolean | `indicator` | Протокол прошивки сумісний із цим адаптером |
| `info.licenseTier` | string | `text` | Активний рівень ліцензії: `free` (моніторинг), `community` (керування реле) або `pro` (+ автономний розклад); порожньо, якщо прошивка без ліцензування |
| `info.licenseTrialDaysLeft` | number | `value` | Залишок пробних днів ліцензії (0 = пробний період не триває) |
| `info.deviceCode` | string | `text` | Код пристрою — вкажіть його під час розблокування, щоб отримати ключ активації |
| `info.licenseControlBlocked` | boolean | `indicator` | Пристрій відхилив команду керування (немає ліцензії на керування) |

**Керування (команди, доступні для запису)**

| Об'єкт | Тип | Роль | Опис |

| `control.point.<n>.open` | boolean (w) | `switch` | Вручну відкрити клапан точки `<n>` |
| `control.group.<g>.active` | boolean (w) | `switch` | Вручну активувати групу `<g>` |

**Точки аерації** (по одному каналу на кожну налаштовану точку, названому за іменем точки)

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `aeration.point.<n>.valveState` | boolean | `indicator` | Клапан відкритий |
| `aeration.point.<n>.active` | boolean | `indicator` | Точка зараз аерує |
| `aeration.point.<n>.buttonOn` | boolean | `indicator` | Кнопка ручного перевизначення активна (лише за налаштованої кнопки) |
