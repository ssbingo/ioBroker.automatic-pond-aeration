## Адаптер automatic-pond-aeration для ioBroker

Цей адаптер **керує системою аерації ставка та контролює її роботу**. Повітряний насос/компресор подає
повітря через клапани (електромагнітні клапани) до **до 8 точок аерації** у ставку. Адаптер перемикає
ці клапани за **розкладом**, за **циклічним колом (round-robin)** або за **груповою програмою** та
захищає насос за допомогою **захисного блокування**: доки насос працює, завжди залишається відкритим
щонайменше один клапан – інакше відкривається **аварійний клапан** і (якщо насос доступний як точка
даних) насос вимикається.

За бажанням він може контролювати **розчинений кисень**, **температуру повітря та води** й **тиск**,
обчислювати **астрономічний час** за вашою **геопозицією**, керувати обладнанням **безпосередньо на
ESP32** (без додаткового екземпляра ioBroker) та призупиняти вибрані точки аерації під час годування,
якщо встановлено [ioBroker.automatic-feeder](https://github.com/ssbingo/ioBroker.automatic-feeder).

> ⚠️ **Статус проєкту: у розробці.** Модель конфігурації та повна модель точок даних готові: адаптер
> перевіряє вашу конфігурацію та відповідно створює (і очищає) усі свої об'єкти. Керуючий рушій,
> апаратні бекенди та функції моніторингу додаються крок за кроком (від віхи до віхи). Для
> промислового використання версія ще не призначена.

## 1. Що робить адаптер

Аерація ставка розподіляє повітря від одного насоса на кілька розпилювачів/повітряних каменів. Які
точки отримують повітря, вирішують **електромагнітні клапани**. Цей адаптер вирішує, **коли**
відкривається кожен клапан:

* **Розклад** – відкривати точку/групу протягом налаштованих часових вікон за днями тижня.
* **Циклічне коло (round-robin)** – по черзі перемикати точки, кожна відкрита протягом
  налаштовуваного часу утримання.
* **Групи** – керувати кількома точками разом; груп **ніколи не може бути більше, ніж точок**.

Клапани й насос можна приводити в дію або через **наявні стани ioBroker** (з будь-якого адаптера, що
надає перемикачі), або **безпосередньо на ESP32** із супутньою прошивкою.

## 2. Концепція безпеки

Повітряний компресор **ніколи не повинен працювати на повністю закриті клапани** (робота «в глухий
кут», dead-heading) – це створює надлишковий тиск і може пошкодити насос. Тому:

* Доки насос працює, **завжди відкритий щонайменше один клапан** (налаштовуваний мінімум).
* Якщо цього не можна гарантувати, **відкривається аварійний клапан** і, якщо насосом можна керувати,
  **насос вимикається**.
* Перемикання клапанів працює за принципом **make-before-break** (наступний клапан відкривається
  раніше, ніж закривається попередній), тож не буває моменту, коли всі клапани закриті.

> 💡 **Рекомендація щодо підключення:** використовуйте **нормально відкритий (NO)** аварійний клапан,
> щоб він відкривався при зникненні живлення (відмовостійкість). Коли обладнання працює на ESP32, те
> саме блокування виконується й локально на пристрої, тож збій мережі або ioBroker не зможе пошкодити
> насос.

## 3. Передумови

* Node.js ≥ 22
* js-controller ≥ 6.0.11, admin ≥ 7.6.20
* Один або кілька клапанів, доступних як стани ioBroker, або ESP32 із супутньою прошивкою.

## 4. Встановлення

Встановіть адаптер з адмінки ioBroker (або, під час розробки, з репозиторію GitHub) і створіть
екземпляр. Відкрийте налаштування екземпляра, щоб налаштувати його.

## 5. Огляд конфігурації

Сторінка налаштувань розширюється разом із віхами. Заплановані розділи: загальні/бекенд, точки
аерації, керування (розклад/round-robin/групи), датчики, астро та розташування, зв'язок із feeder,
безпека та сповіщення. Повний опис див. у [PROJECT_PLAN.md](../../PROJECT_PLAN.md).

## 6. Об'єкти / точки даних

Адаптер створює свої точки даних із вашої конфігурації. Заповнювачі: `<n>` = індекс точки аерації
(0–7), `<g>` = індекс групи. Об'єкти, позначені **(w)**, — це команди, доступні для запису; усі інші
— доступні лише для читання значення стану, які оновлює адаптер.

**Загальне**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `info.connection` | boolean | `indicator.connected` | Адаптер працює / конфігурація дійсна |
| `info.backend` | string | `text` | Активний апаратний бекенд (`iobroker` або `esp32`) |
| `info.activeMode` | string | `text` | Поточний режим роботи |

**Керування (команди, доступні для запису)**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `control.enabled` | boolean (w) | `switch.enable` | Головне ввімкнення |
| `control.mode` | string (w) | `text` | Режим роботи: `auto`, `manual` або `off` |
| `control.allOff` | boolean (w) | `button` | Закрити всі клапани |
| `control.point.<n>.open` | boolean (w) | `switch` | Вручну відкрити клапан точки `<n>` |
| `control.group.<g>.active` | boolean (w) | `switch` | Вручну активувати групу `<g>` |

**Точки аерації** (по одному каналу на кожну налаштовану точку, названому за іменем точки)

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `aeration.point.<n>.valveState` | boolean | `indicator` | Клапан відкритий |
| `aeration.point.<n>.active` | boolean | `indicator` | Точка зараз аерує |
| `aeration.point.<n>.runtimeTodaySec` | number | `value` | Наробіток за сьогодні (секунди) |
| `aeration.point.<n>.runtimeTotalH` | number | `value` | Загальний наробіток (години, для обслуговування) |
| `aeration.point.<n>.lastChange` | number | `value.time` | Позначка часу останньої зміни клапана |
| `aeration.point.<n>.error` | string | `text` | Остання помилка для цієї точки |

**Групи**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `groups.<g>.members` | string | `json` | Індекси точок-учасників |
| `groups.<g>.active` | boolean | `indicator` | Група зараз активна |

**Безпека**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `safety.interlockActive` | boolean | `indicator.alarm` | Захисне блокування зараз активне |
| `safety.emergencyValve` | boolean | `indicator` | Аварійний клапан відкритий |
| `safety.pumpRunning` | boolean | `indicator` | Насос працює |
| `safety.openValveCount` | number | `value` | Кількість відкритих клапанів |
| `safety.lastTripReason` | string | `text` | Причина останнього спрацювання блокування |

**Датчики** (створюються лише за ввімкненого відповідного моніторингу)

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `sensors.oxygen` | number | `value` | Розчинений кисень (mg/l) |
| `sensors.oxygenSaturation` | number | `value` | Насичення киснем (%) |
| `sensors.oxygenAlarm` | boolean | `indicator.alarm` | Кисень нижче нижнього порога |
| `sensors.airTemperature` | number | `value.temperature` | Температура повітря (°C) |
| `sensors.waterTemperature` | number | `value.temperature` | Температура води (°C) |
| `sensors.pressure` | number | `value.pressure` | Тиск у системі (bar) |
| `sensors.pressureAlarm` | boolean | `indicator.alarm` | Тиск поза діапазоном |

**Астрономія та розташування**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `astro.sunrise` / `astro.sunset` / `astro.solarNoon` | string | `text` | Час сонця для розташування |
| `astro.isNight` | boolean | `indicator` | Зараз ніч |
| `location.latitude` / `location.longitude` | number | `value.gps.*` | Визначені координати |
| `location.resolvedAddress` | string | `text` | Визначена адреса |

**Зв'язок із feeder** (створюється лише за ввімкненого зв'язку з feeder)

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `feeder.pauseActive` | boolean | `indicator` | Аерацію призупинено на час годування |
| `feeder.pauseUntil` | number | `value.time` | Пауза активна до |
| `feeder.lastFeedStart` | number | `value.time` | Останній початок годування |

**Статистика**

| Об'єкт | Тип | Роль | Опис |
|--------|-----|------|------|
| `statistics.compressorRuntimeTodayH` | number | `value` | Наробіток компресора за сьогодні (години) |
| `statistics.switchCyclesToday` | number | `value` | Циклів перемикання клапанів за сьогодні |

Коли точку, групу або датчик видаляють із конфігурації, їхні об'єкти очищуються автоматично.
